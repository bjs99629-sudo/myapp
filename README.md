# 📅 스마트 출석체크 시스템 (Attendance Tracker)

---

## 🌟 주요 기능 및 특징

1. **간편한 출석 체크 (Check-in)**
   - 이름과 날짜, 시간을 입력해 간편하게 출석을 기록합니다.
   - 당일 중복 출석은 자동으로 차단되어 데이터 중복을 방지합니다.
2. **지각 자동 판별**
   - 관리자가 설정한 출석 기준 시간(예: 09:00)을 초과하여 체크인 시 자동으로 **지각** 처리됩니다.
3. **실시간 대시보드 및 통계**
   - 누적 출석 횟수, 지각 횟수, 정상 출석률(%) 등 주요 지표를 요약 카드로 제공합니다.
   - 일자별 출석/지각 추이 막대그래프 및 개인별 출석률 성실도 랭킹 차트를 실시간으로 시각화합니다.
4. **관리 콘솔 (Admin Console)**
   - 비밀번호 보안이 적용된 관리 공간에서 출석 기준 시간과 관리자 암호를 변경할 수 있습니다.
   - 전체 데이터를 조회하고, 필요 시 CSV 파일로 데이터를 백업하거나 초기화할 수 있습니다.
5. **유연한 하이브리드 데이터베이스 (Google Sheets & Local CSV)**
   - 구글 시트 연동 설정이 없는 상태에서 실행하면, 로컬 CSV(`attendance_db.csv`)로 즉각 동작하는 **로컬 폴백 모드**를 지원합니다.
6. **프리미엄 반응형 UI/UX**
   - 세련된 네온 다크 그라디언트 테마와 카드 레이아웃, 모바일 지원 반응형 디자인이 적용되었습니다.

## 🚀 로컬 실행 방법

1. **저장소 복제 및 이동**
   ```bash
   git clone <your-github-repo-url>
   cd <repo-name>
   ```

2. **의존성 라이브러리 설치**
   ```bash
   pip install -r requirements.txt
   ```

3. **애플리케이션 실행**
   ```bash
   streamlit run app.py
   ```
   *실행 시 구글 시트 연동 정보가 없어도 로컬 파일 모드로 자동 시작되어 즉시 테스트할 수 있습니다.*

## 🔗 구글 스프레드시트 연동 가이드

안전하게 구글 스프레드시트를 DB로 활용하려면 아래의 순서대로 API 설정을 완료해야 합니다.

### 1단계: Google Sheets API & 서비스 계정 설정
1. [Google Cloud Console](https://console.cloud.google.com/)에 접속하여 새 프로젝트를 만듭니다.
2. **API 및 서비스 > 라이브러리** 메뉴에서 **Google Sheets API**와 **Google Drive API**를 검색하여 활성화합니다.
3. **API 및 서비스 > 사용자 인증 정보**로 이동하여 **사용자 인증 정보 만들기 > 서비스 계정**을 클릭합니다.
4. 서비스 계정을 생성하고 생성된 계정의 **키(Key)** 탭에서 **새 키 만들기 > JSON**을 선택해 다운로드합니다. (이 JSON 파일에는 중요한 인증 키가 포함되어 있습니다.)

### 2단계: 스프레드시트 준비 및 권한 부여
1. [Google Sheets](https://docs.google.com/spreadsheets/)에서 출석부로 사용할 새 시트를 생성합니다.
2. 첫 번째 행에 다음 컬럼명을 입력합니다:
   `날짜`, `이름`, `체크인 시간`, `상태`, `비고`
3. 스프레드시트 우측 상단의 **공유** 버튼을 누르고, **1단계에서 복사한 서비스 계정의 이메일 주소**(`...@...iam.gserviceaccount.com`)를 추가하여 **편집자** 권한을 부여합니다.

### 3단계: Streamlit Secrets 설정
스프레드시트 정보와 인증 정보를 Streamlit에 주입해야 합니다. **인증 키 파일은 절대로 GitHub에 업로드하지 마세요!**
